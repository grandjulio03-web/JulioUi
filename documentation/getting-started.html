Quick Start

Get Air running in under 2 minutes. No build tool required for prototyping — just link the CSS and go.

1. Include CSS

Link the compiled stylesheet. One file covers everything.

HTML
<head>
  <!-- Air Framework CSS -->
  <link rel="stylesheet" href="path/to/dist/my-framework.css">
</head>

2. Add dark mode flash prevention

Place this script before your CSS link to prevent a white flash on dark-mode users.

HTML — in <head>, before CSS
<script>
  try {
    var t = localStorage.getItem('air-theme');
    if (t) document.documentElement.setAttribute('data-theme', t);
    else if (window.matchMedia('(prefers-color-scheme: dark)').matches)
      document.documentElement.setAttribute('data-theme', 'dark');
  } catch(e) {}
</script>

3. Include JS modules

Each JS file is independent. Include only what you use — before </body>.

air-nav.jsNav auto-panel, indicator, dropdowns, scroll behaviorair-nav, air-side

<script src="src/js/air-nav.js"></script>
<script src="src/js/air-modal.js"></script>
<script src="src/js/air-tabs.js"></script>
<script src="src/js/air-animate.js"></script>
<script src="src/js/air-theme.js"></script>

4. Build from source (optional)

Use SASS to compile and customize the framework.

Terminal
# Install dev dependency
npm install

# Compile SCSS → CSS (expanded, for development)
npm run build-css

# Compile SCSS → CSS (minified, for production)
npm run minify-css

5. Hello Air

Online
HTML
<button class="air-btn ace">Hello Air</button>
<button class="air-btn beta ghost">Ghost</button>
<button class="air-btn gamma glow">Glow</button>
<span   class="air-badge delta soft dot">Online</span>
Pattern: Air uses component tone variant size — always separated as individual classes. air-btn ace glow lg — that's it. No prefix spam, no double dashes, no BEM modifiers.
