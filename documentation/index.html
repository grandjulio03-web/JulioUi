What makes Air different

✦

Air's own vocabulary

Write shelf not d-flex align-items-center. Write brand not navbar-brand. Human words, CSS power.

⚡

Chromatic shadows

Shadows carry the button's own color, not plain black. 6 button variants, spring physics on every interaction.

🌙

Auto dark mode

One attribute on <html>. CSS custom property cascade handles every component. System preference detected. No flash.

🎯

Scroll animations built in

data-animate="rise" on any element. IntersectionObserver adds .awake when it enters view. Zero config.

🌊

Unique backgrounds

Mesh gradients, aurora, glows, noise grain, dot grids, light beams — all pure CSS, one class. Bootstrap has nothing like this.

📱

Auto mobile panel

Write nav links once. JS auto-clones them into the mobile slide panel. Bootstrap makes you write everything twice.

5-Tone color system

Every component accepts a tone class. Mix tones with variants freely — air-btn ace glow, air-card beta dark.

Quick start

Include the compiled CSS and any JS modules you need. No build tool required.

HTML 
<!-- 1. CSS (one file) -->
<link rel="stylesheet" href="dist/my-framework.css">

<!-- 2. JS modules (include only what you use) -->
<script src="src/js/air-nav.js"></script>
<script src="src/js/air-modal.js"></script>
<script src="src/js/air-tabs.js"></script>
<script src="src/js/air-animate.js"></script>
<script src="src/js/air-theme.js"></script>
<script src="src/js/air-spacing.js"></script>

<!-- 3. Write Air HTML -->
<button class="air-btn ace glow">Get started</button>
<div    class="air-bg mesh">Hero section</div>
<div    data-animate="rise">Wakes on scroll</div>
Build from source: Run npm run build-css to compile SCSS → CSS. Run npm run minify-css for a minified production build. No other build step needed — JS files are vanilla, no bundler required.
