Light & Dark Theme

One toggle. Every component switches. CSS custom properties cascade from <html data-theme="dark">. Preference saved to localStorage. System preference auto-detected.

Current: light
How it works — three steps
Step 1

Custom properties cascade

All Air colors are CSS custom properties. Dark mode overrides them at [data-theme="dark"] on <html>.

Step 2

Zero class changes on components

You never touch component markup. Toggle one attribute on the root, every card, input, nav, badge switches instantly.

Step 3

System preference + localStorage

On first visit, reads prefers-color-scheme. After user toggles, preference is saved. No flash on next load.

Typography

Typography in dark mode

Body text uses var(--air-700) which maps to the dark-tinted neutral in dark mode. Headings use var(--air-900) for maximum contrast.

All type tokens are CSS custom properties. Toggle the theme and text naturally adapts.
  • Token-driven: no hardcoded colors
  • Smooth transition: 380ms ease
  • Neutral scale: 10 stops, green-tinted in dark

Navigation (the top nav follows theme)

The live nav at the top of this page uses air-nav ace without any explicit dark modifier. When data-theme="dark" is applied to <html>, the nav automatically switches via CSS custom property cascade.

Write once, theme for free
/* 1. Include air-theme.js (adds data-theme toggle) */
<script src="air-theme.js"></script>

/* 2. Add toggle button anywhere */
<button class="air-theme-btn" data-theme-toggle>
  <svg class="icon-moon">...</svg>
  <svg class="icon-sun">...</svg>
</button>

/* 3. Write components normally — no dark: prefix spam */
<div class="air-card ace">...</div>

/* API */
Air.theme.toggle()        // switch light/dark
Air.theme.set('dark')     // force dark
Air.theme.get()           // returns 'light' | 'dark'
Air.theme.onChange(fn)    // subscribe to changes
