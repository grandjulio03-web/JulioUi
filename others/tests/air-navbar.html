Air Framework

Navigation — Air's own words

Air doesn't borrow Bootstrap's vocabulary. No navbar-brand, no nav-item, no ul>li. Every part has a name a human actually uses.

brand · links · link · end · menu fly (dropdown) floor · tab air-side · path · mark air-trail · crumb
Air's vocabulary — simple words, powerful meaning
air-nav

The nav bar itself. Sticky rail by default. Add float or floor for other types.

.brand

Your logo or site name. Direct child of air-nav. Just an <a> tag.

.links

Holds all nav links. A <div> — no ul, no li required.

.link

One nav link. Add .active for current page. Add data-fly="id" for a dropdown.

.end

The right-side zone — your CTAs, login buttons, avatars. Automatically hidden on mobile.

.menu

Hamburger button. JS auto-builds the mobile panel from your .links — no duplication.

.fly

A dropdown panel. Triggered by data-fly="id". Add .wide for mega layout.

.fly-row

One item inside a .fly. Has icon, title, description — grid or stacked.

air-nav.float

Floating pill nav. Fixed, centered, rounded. The nav IS the pill — no inner wrapper.

air-nav.floor

Bottom tab bar. iOS-native feel. Use .tab inside, no links/end needed.

air-side

Sidebar nav. Parts: .mark (brand), .section (label), .path (link), .tail (badge).

air-trail

Breadcrumb. Use .crumb for each step. The separator is pure CSS — no extra HTML.
